## Overview

Mindlap Social Scheduler uses OpenAI's GPT-4o-mini model to help you create compelling social media content. From generating posts from simple prompts to improving existing content, AI can save time and boost engagement.

## Generate a Post from a Prompt

Create a complete social media post from a simple idea.

<CodeGroup>
```bash cURL
curl -X POST http://localhost:5000/api/ai/generate/post \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Announce our new AI-powered customer service feature",
    "platform": "linkedin",
    "tone": "professional",
    "audience": "business professionals",
    "contentType": "promotional",
    "includeHashtags": true,
    "maxLength": 300
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:5000/api/ai/generate/post', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    prompt: 'Announce our new AI-powered customer service feature',
    platform: 'linkedin',
    tone: 'professional',
    audience: 'business professionals',
    contentType: 'promotional',
    includeHashtags: true,
    maxLength: 300
  })
});

const { content, hashtags } = await response.json();
```
</CodeGroup>

### Generation Parameters

<ParamField path="prompt" type="string" required>
  Your content idea or topic
</ParamField>

<ParamField path="platform" type="string" default="twitter">
  Target platform: `twitter`, `linkedin`, `instagram`, `youtube`
</ParamField>

<ParamField path="tone" type="string" default="professional">
  Writing style: `professional`, `casual`, `friendly`, `enthusiastic`, `humorous`, `informative`
</ParamField>

<ParamField path="audience" type="string">
  Target audience description (e.g., "business professionals", "developers", "marketing managers")
</ParamField>

<ParamField path="contentType" type="string" default="general">
  Content category: `general`, `educational`, `promotional`, `engagement`, `announcement`
</ParamField>

<ParamField path="includeHashtags" type="boolean" default="true">
  Whether to include relevant hashtags
</ParamField>

<ParamField path="maxLength" type="number">
  Maximum character count (respects platform limits)
</ParamField>

## Improve Existing Content

Enhance your existing posts for better engagement.

```bash

  -H "Authorization: Bearer YOUR_TOKEN" \

    "originalContent": "We have a new feature. Check it out.",
    "platform": "twitter",
    "improvements": ["engagement", "clarity", "cta"]
```

## Generate Bulk Posts

Create multiple posts for a campaign in one request.

```bash
curl -X POST http://localhost:5000/api/ai/generate/bulk-posts \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "campaign_theme": "Mindlap product launch week",
    "platforms": ["twitter", "linkedin"],
    "post_count": 5,
    "content_types": ["educational", "promotional", "engagement"],
    "tone": "professional",
    "audience": "tech entrepreneurs and business owners"
  }'
```

## Smart Scheduling with AI

Get AI recommendations for the best times to post.

```bash
curl -X POST http://localhost:5000/api/ai/optimize/scheduling \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "contentType": "educational",
    "platforms": ["linkedin", "twitter"],
    "audience": "business professionals",
    "timezone": "America/New_York",
    "priority": "high"
  }'
```

**Response:**
```json
{
  "recommendation": "For business professionals on LinkedIn and Twitter, optimal posting times are weekday mornings (9-10 AM) and lunch hours (12-1 PM) EST when engagement is highest.",
  "optimal_times": ["9:00 AM", "12:00 PM", "5:00 PM"],
  "best_days": ["tuesday", "wednesday", "thursday"],
  "frequency": "3_times_week",
  "platforms": ["linkedin", "twitter"],
  "timezone": "America/New_York"
}
```

### Then Schedule Posts at Optimal Times

```bash
curl -X POST http://localhost:5000/api/ai/smart-schedule \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "posts": [
      {
        "content": "Post 1 content...",
        "platforms": ["linkedin"]
      },
      {
        "content": "Post 2 content...",
        "platforms": ["twitter"]
      }
    ],
    "audience": "business professionals",
    "timezone": "America/New_York"
  }'
```

The AI will automatically distribute your posts across optimal times.

## Best Practices for AI Generation

<Steps>
  <Step title="Be Specific in Prompts">
    Better: "Announce our new AI customer service feature that reduces response times by 80%"

    Worse: "New feature announcement"
  </Step>

  <Step title="Define Your Audience">
    Specify who you're targeting:
    - "C-suite executives in tech"
    - "Small business owners"
    - "Developers and engineers"
  </Step>

  <Step title="Choose the Right Tone">
    Match your brand personality:
    - Professional for B2B LinkedIn
    - Casual for Twitter engagement
    - Friendly for customer support
  </Step>

  <Step title="Review and Edit">
    AI-generated content is a starting point:
    - Add personal touches
    - Verify facts and claims
    - Ensure brand alignment
    - Check for errors
  </Step>

  <Step title="Test Different Variations">
    Generate multiple versions and A/B test:
    - Try different tones
    - Experiment with lengths
    - Test various hashtags
  </Step>
</Steps>

## Cost Considerations

<Info>
  AI features use OpenAI's API, which has usage costs. Monitor your API usage in the OpenAI dashboard.
</Info>

**Typical Costs:**
- Single post generation: ~200-400 tokens (~$0.001-0.002)
- Content improvement: ~300-500 tokens (~$0.001-0.003)
- Bulk generation (5 posts): ~1,000-2,000 tokens (~$0.005-0.010)
- Template generation: ~400-600 tokens (~$0.002-0.003)

Model: **gpt-4o-mini** (cost-effective option)

## Troubleshooting

<AccordionGroup>
  <Accordion title="AI generation failed">
    **Error:** "Failed to generate content"

    **Solutions:**
    - Verify OpenAI API key is valid
    - Check API credit balance
    - Test connection: `node test_openai.js`
    - Review OpenAI API status
  </Accordion>

  <Accordion title="Generated content is too short/long">
    **Issue:** Content doesn't meet expectations

    **Solutions:**
    - Adjust `maxLength` parameter
    - Be more specific in prompt
    - Try different `contentType` values
    - Regenerate with modified parameters
  </Accordion>

  <Accordion title="Inappropriate content generated">
    **Issue:** AI generated off-brand content

    **Solutions:**
    - Provide more context in prompt
    - Specify tone and audience clearly
    - Use content improvement to refine
    - Add brand guidelines to prompt
  </Accordion>
</AccordionGroup>
